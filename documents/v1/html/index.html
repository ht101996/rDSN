rDSN Documentation

Robust Distributed System Nucleus (rDSN) is a modular framework for quickly building robust and high-performance distributed systems. The goal is to enable independent development and seamless integration of applications, frameworks, local runtime libraries, and tools, so that they can transparently benefit each other. The project was originally developed for Microsoft Bing, and now has been adopted in production both inside and outside Microsoft.

What can be done with rDSN?

  • an enhanced event-driven RPC library such as libevent, Thrift, and GRPC (see [tutorial]() for what's better).
  • a production Paxos framework to quickly turn a local component (e.g., rocksdb) into a online service with replication, partition, failure recovery, and reconfiguration supports ([tutorial]()).
  • a scale-out and fail-over framework for stateless services such as Memcached ([tutorial]()).
  • more as you can imagine.

What is great/novel about rDSN?

  • microkernel architecture: applications, frameworks (e.g., replication, scale-out, fail-over), local runtime libraries (e.g., network libraries, locks), and tools are all pluggable modules into a microkernel to enable independent development and seamless integration (therefore all modules are reusable and transparently benefit each other)
  • first-class dev&op support: dedicated tool API for tool development; built-in plugged tools for understanding, testing, debugging, and monitoring the upper applications and frameworks
  • configurable runtime: tailor the module instances and their mapping on demand with controllable system complexity (e.g., run all nodes in one simulator for testing, allocate CPU resources appropriately for avoiding resource contention, debug with progressively added system compelxity)
